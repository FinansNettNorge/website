Technology

FNN Core Network Architecture

FNN's core network is built on Extreme IP-Fabric technology, a modern, scalable foundation for high-performance networking. The network infrastructure is distributed across multiple geographical locations within Oslo, providing redundancy, resilience, and continuous service availability for the Norwegian financial sector.

IP-Fabric Technology Advantages

The IP-Fabric architecture offers significant benefits for financial services networks:

  • Scalability: Easily accommodates growing network demands without degradation of performance
  • Redundancy & Resilience: Multi-location deployment ensures network continuity and eliminates single points of failure
  • Low Latency: Optimized for high-speed financial transactions and data exchange
  • Automation: Reduces operational complexity and human error in network management
  • Enhanced Security: Built-in isolation and encryption capabilities protect sensitive financial data
  • Stealth Architecture: Network operates as a closed, private infrastructure invisible to external threats

Customer Connectivity – Dark Fibre from GlobalConnect

Customer-Premises Equipment (CPE) connects to the FNN IP-Fabric core network via dark fibre provided by GlobalConnect. This direct, dedicated connection ensures:

  • Guaranteed bandwidth with no contention from external users
  • Complete isolation of customer traffic from other networks
  • Predictable, low-latency connectivity suitable for time-sensitive financial operations
  • Flexibility to scale capacity as customer requirements grow

Security Architecture

FNN operates as a closed, private network with dedicated connectivity and service isolation. This architecture provides:

  • Network isolation: Customers access FNN through dedicated dark fibre connections, not shared public infrastructure
  • Service separation: Full service separation based on SPB (IEEE 802.1aq) ensures traffic isolation between customers
  • Layer 2 segregation: L2 VPN technology with non-routable network addresses in the core network
  • Layer 3 segregation: L3 VPN with traffic separation and routing isolation using VRF (Virtual Routing Forwarding)

This multi-layered approach ensures that customer data remains isolated and protected within the FNN network infrastructure.
